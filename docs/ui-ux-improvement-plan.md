# UI / UX 改善 実装計画

## 1. 背景

現状の UI は以下の点で基礎品質が高い。

- 余白設計とカード構成が安定している
- モバイル前提の構成ができている
- バッジ、進捗バー、カレンダーなどの部品は破綻していない

一方で、以下の課題がある。

- 生徒一覧から詳細までの導線が1アクション多い（Sheet を挟む）
- 設定画面に更新履歴・使い方がなく、講師の自己解決手段がない
- 選択日の授業記録パネルの視認性が弱い
- デザイントークンとカスタムクラスの混在（`text-text-muted` vs `text-muted-foreground`）
- ボタン文言・補助情報の文字サイズにばらつきがある

## 2. 方針

改善は「導線の短縮」と「既存画面の視認性向上」に絞る。
大きな構造変更（ダッシュボード推薦エンジン、記録入力の再設計、生徒詳細の表示順変更）は現時点では行わない。

### やらないことと理由

| 項目 | 見送り理由 |
|------|-----------|
| ダッシュボード「今やること」追加 | ユーザーは1〜2名の講師、生徒数も数十名規模。次にやることは講師の頭にある。推薦が的外れだと逆効果。生徒50名超で再検討 |
| 生徒詳細の表示順変更（案A） | カレンダー最上部は「前回の日付と内容」を一目で見る配置として機能している。「次にやること」の機械的推薦を主役にすると、複数教科の生徒で邪魔になるリスク |
| 記録入力の構造再設計 | `renderManualSelectors` は既に写真/手動で共有済み。「別アプリのように見える」は実態と乖離。`aiLockRef` 等の制御が絡み、再設計コストが見合わない |

## 3. 実装フェーズ

### Phase 1: 導線の短縮（工数: 小）

#### 1-1. 生徒一覧の1タップ詳細化

**対象**: `students/student-list-client.tsx`, `students/page.tsx`

**現状**: カードタップ → Sheet（最近の記録8件）→「詳細を見る」ボタン → 詳細ページ。
Sheet の中身は詳細ページと重複しており、挟む価値がない。

**変更内容**:
- カード全体を `/students/[id]` への `Link` に変更
- Sheet を削除
- カード内の情報は現状維持（名前・学年・科目・進捗）

**完了条件**: 一覧から詳細までが1タップになる

---

### Phase 2: 設定画面の強化（工数: 小）

#### 2-1. 更新履歴・使い方・お困りの場合を追加

**対象**: `settings/page.tsx`

**現状**: アカウント情報とバージョン表示のみ。機能追加時に LINE 告知だけに依存している。

**変更内容**:
- 「更新履歴」セクション追加 — `CHANGELOG.md` の内容をハードコード or 読み込みで表示
- 「使い方」セクション追加 — 主要機能の簡易説明（写真で記録する流れ、生徒の追加方法など）
- 「お困りの場合」セクション追加 — 管理者への連絡先

**実装方針**:
- CHANGELOG.md と二重管理を避けるため、設定画面側はバージョンごとの要約のみ
- 使い方は静的コンテンツ。凝った仕組みは不要

**完了条件**: 後から入った講師が設定画面だけで基本操作を理解できる

---

### Phase 3: 生徒詳細の視認性改善（工数: 小）

#### 3-1. 選択日の授業記録パネルの強化

**対象**: `students/[id]/page.tsx`

**現状**: 日別記録は表示されるが、他のカードと同じ視覚的重みで埋もれている。

**変更内容**:
- 選択日パネルのヘッダーを強調（背景色 or ボーダー）
- 各レコード行の情報密度を上げる（コメント表示、score_source 表示など）
- 編集・削除ボタン（RecordActions）の視認性確認

**変更しないもの**:
- カレンダーの位置（最上部を維持）
- 学習ペース・科目別進捗の表示順

**完了条件**: 日付タップ後に「何をやったか」が一目でわかる

---

### Phase 4: デザインシステム統一（工数: 中）

#### 4-1. トークン統一

**対象**: 全画面（画面ごとに段階的に適用）

**方針**:
- 新規実装は `foreground / muted-foreground / background / card / border / primary` に統一
- 旧クラス（`text-text-muted`, `bg-surface`, `text-text`）を画面単位で置換
- 一括置換はしない

#### 4-2. 文字サイズルール

| 用途 | サイズ |
|------|--------|
| 主要本文 | `text-sm`（14px） |
| 補助情報 | `text-xs`（12px） |
| バッジ・タイムスタンプ | `text-[11px]` まで許容 |
| 10px 以下 | 原則使わない |

#### 4-3. ボタン文言の統一

- 記録入力画面内のボタン文言を棚卸しし、同じ操作には同じ文言を使う
- 「保存」「記録する」「登録する」の使い分けを明確にする

**完了条件**: 新規画面追加時にどのトークン・サイズを使うか迷わない

---

## 4. 優先順位まとめ

| 順位 | 項目 | 工数 | 効果 |
|------|------|------|------|
| 1 | 生徒一覧の1タップ詳細化 | 小 | 大 |
| 2 | 設定画面に更新履歴・使い方を追加 | 小 | 中 |
| 3 | 選択日の授業記録パネルの視認性強化 | 小 | 中 |
| 4 | デザイントークンの段階統一 | 中 | 中（保守性） |

## 5. 将来の検討事項

以下は現時点では見送るが、条件が変わったら再検討する。

| 項目 | 再検討の条件 |
|------|-------------|
| ダッシュボード「今やること」 | 生徒数が50名を超えた / 講師が3名以上になった |
| 生徒詳細の表示順変更 | 講師から「前回何やったか探しにくい」等のフィードバック |
| 記録入力の構造再設計 | 写真/手動の混乱に関する具体的な操作ミス報告 |
| エラー・空状態の設計 | 生徒0名・AI連続失敗・ネットワーク断時の体験改善 |
