# 設定画面内案内・更新履歴・バージョン管理 提案

## 1. 目的

アプリを運用していく中で、以下をユーザーがすぐ確認できる状態を作る。

- いつ、どの機能が追加・変更されたか
- よく使う機能をどう操作すればよいか
- 困った時にどこを見ればよいか
- 現在使っているアプリのバージョンが何か

## 2. 結論

最初にやるべきことは、**アプリ内の設定画面に簡易案内を入れること**。  
その上で、詳細な内容は**別マニュアル**として管理するのが最も実用的。

役割分担は以下とする。

### 設定画面に置くもの

- 更新履歴
- よく使う機能の簡単な説明
- 困った時の案内
- 現在のバージョン表示

### 別マニュアルに置くもの

- 初期設定手順
- 生徒登録から授業記録までの一連の流れ
- 管理者向けの運用ルール
- トラブル対応集
- 詳細なFAQ

## 3. なぜこの構成がよいか

### 設定画面に簡易案内を置く理由

- ユーザーは困った時にまずアプリ内を見る
- 機能追加の周知漏れを減らせる
- LINE や口頭連絡に依存しなくて済む
- 画面を見ながらすぐ確認できる

### 別マニュアルを分ける理由

- アプリ内に情報を詰め込みすぎると読まれない
- 管理者向けと講師向けで必要な情報量が違う
- 詳細な運用ルールは後から追記・改訂しやすい

## 4. 設定画面への追加提案

現状の設定画面は以下の2ブロックのみ。

- アカウント情報
- バージョン

これに加えて、以下の3ブロックを追加することを推奨する。

### 4.1 更新履歴

目的:
- 新機能や修正内容を短く周知する

表示イメージ:

```txt
更新履歴
2026-04-10 v0.2.0
- 授業記録の編集機能を追加
- 授業記録の削除機能を追加
- 削除時に答案画像も連動削除
```

方針:
- 最新3〜5件だけ表示
- 詳細は「マニュアルを見る」リンクへ誘導

### 4.2 使い方

目的:
- よく使う操作の入口を短く説明する

候補:
- 授業記録の登録方法
- 記録を間違えた時の編集・削除方法
- 保護者レポートの出力方法

表示イメージ:

```txt
使い方
・授業記録を修正したい時
  生徒詳細ページ → カレンダーで日付選択 → 記録右端の「⋮」から編集/削除

・パスワードを忘れた時
  ログイン画面の「パスワードを忘れた方」から再設定
```

### 4.3 困ったとき

目的:
- 問い合わせ前に自己解決しやすくする

候補:
- 三点リーダーが出ない
- ログインできない
- 間違って登録した記録を消したい

表示イメージ:

```txt
困ったとき
・編集ボタンが出ない
  生徒詳細 → 日付選択後の授業記録一覧から操作できます

・ログインできない
  メールアドレス確認後、パスワードリセットをお試しください
```

## 5. バージョン管理の提案

### 5.1 バージョン規則

`SemVer` で管理する。

- `0.1.0` 初期リリース
- `0.1.1` バグ修正
- `0.2.0` 新機能追加
- `0.3.0` 操作フローに影響する追加機能
- `1.0.0` 基本運用が安定した段階

### 5.2 今回の変更に対する推奨

授業記録の編集・削除機能追加は**新機能追加**なので、  
`0.1.0` から `0.2.0` への更新が妥当。

### 5.3 管理方法

管理元は `package.json` の `version` を正とする。

画面表示はハードコードせず、`package.json` の version を参照する。

対象:
- ログイン画面
- 設定画面

### 5.4 リリース時に更新するもの

毎回以下をセットで更新する。

1. `package.json` の version
2. `package-lock.json`
3. `CHANGELOG.md`
4. 設定画面の更新履歴表示データ

## 6. CHANGELOG 運用提案

リポジトリ直下に `CHANGELOG.md` を追加する。

例:

```md
# Changelog

## 0.2.0 - 2026-04-10
- 授業記録の編集機能を追加
- 授業記録の削除機能を追加
- 削除時に答案画像もあわせて削除

## 0.1.0 - 2026-04-01
- 初期リリース
- 生徒管理
- 授業記録登録
- 保護者レポート出力
```

## 7. 実装優先順位

### Phase 1: まずやる

- 設定画面に「更新履歴」追加
- 設定画面に「使い方」追加
- バージョン表示を `package.json` 基準に統一
- `CHANGELOG.md` 追加

### Phase 2: 必要に応じて

- 設定画面に「困ったとき」追加
- 詳細マニュアルへのリンク追加
- 管理者向けマニュアル整備

### Phase 3: 将来的に

- 更新履歴を管理画面から追加できるようにする
- 初回ログイン時に新機能案内を表示する

## 8. 実務上のおすすめ

まずは大きく作りすぎず、以下の最小構成で始めるのが良い。

- 設定画面に `更新履歴`
- 設定画面に `使い方`
- リポジトリに `CHANGELOG.md`
- バージョンを `0.2.0` に更新

これで、
- ユーザー向けの案内
- 運営側の変更履歴
- バージョンの整合性

の3点が一気に改善される。

## 9. 補足

LINE などで新機能を告知する運用は今後も有効だが、  
それだけに頼ると「後から入った人」「読み返したい人」が追えない。

そのため、運用方針としては以下が望ましい。

- 初回告知: LINE
- 常設案内: 設定画面
- 詳細説明: マニュアル / CHANGELOG
